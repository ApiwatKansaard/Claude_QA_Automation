# Claude QA Automation — Playwright Test Suite

This is the Playwright automation repo for the QA team at Amity Solutions.
It is a companion to **Claude_QA_Agent** (the AI QA agent repo at `../Claude_QA_Agent/`).

## Project structure

```
src/
├── config/env.config.ts        # Multi-environment config
├── fixtures/test-fixtures.ts   # Custom Playwright fixtures
├── helpers/                    # Auth, API, cleanup, data helpers
├── pages/                      # Page Object Model classes
└── types/                      # TypeScript type definitions

tests/
├── auth.setup.ts               # Authentication setup
├── e2e/                        # UI end-to-end tests (one folder per feature)
└── api/                        # API integration tests

selectors/                      # JSON selector maps (generated by /auto-inspect)
test-data/                      # JSON test data files
environments/                   # Per-environment .env files (.env.dev, .env.staging, .env.prod)
```

## Key conventions

- **Page Object Model**: All selectors live in `src/pages/`. Never put selectors in test files.
- **Tags**: Every test must have `@smoke`, `@sanity`, `@regression`, or `@P1`/`@P2` tags.
- **Test independence**: Each test must run standalone — no order dependencies.
- **Selector hierarchy**: `data-testid` > ARIA roles > text > CSS selectors.
- **No hardcoded data**: Use `test-data/*.json` and fixtures.
- **Feature branches**: `auto/{sprint}/{feature}` pattern for isolation.

## Running tests

```bash
# Install dependencies
npm install

# Run smoke tests
npx playwright test --grep @smoke

# Run all tests
npx playwright test

# Run specific project (e2e, api, mobile, firefox)
npx playwright test --project=e2e
```

## Environments

Set environment via `ENV` variable (dev/staging/prod). Default: dev.
Copy `.env.example` to `environments/.env.dev` and fill in credentials.

## AI agent integration

The **playwright-automator** agent (in Claude_QA_Agent) writes test files to this repo.
When working with the agent:
- `/auto-generate` — generates tests from TestRail CSV into `tests/`
- `/auto-inspect [URL]` — generates selector maps into `selectors/`
- `/auto-run` — runs tests via Playwright CLI
- `/auto-review` — reviews code quality before merge
